## Project Summary: `rizum-blender-clip-reload`

This is a **Blender addon** that reads and renders Clip Studio Paint (`.clip`) files. The core engine (`clip_loader.py`, ~5000 lines) is a from-scratch reverse-engineered `.clip` file parser and rasterizer â€” it doesn't depend on CSP being installed. The GUI integration lives in `clip_studio_importer/` and is packaged as `clip_studio_importer.zip`.

The adjacent project `rizum-clip-studio-paint` is an **IDA Pro workspace** used for native binary analysis of CSP's actual rendering engine to inform the Python reimplementation.

## Architecture & What's Been RE'd

### 1. Container Format (solved)
- **Chunk format**: `CSFCHUNK` magic â†’ `CHNKHead` â†’ `CHNKExta` blobs â†’ `CHNKSQLi` (embedded SQLite).
- Each `CHNKExta` has an ASCII `external_id` and zlib-compressed tile blocks.
- Custom block parser handles the `Bl` marker frame + `BlockDataBeginChunk`/`BlockStatus`/`BlockCheckSum` three-block structure.

### 2. SQLite Schema (mostly mapped)
- `Canvas`, `Layer`, `Mipmap`, `MipmapInfo`, `Offscreen`, `VectorObjectList`, `LayerThumbnail` etc.
- Layer types: raster (`1`), raster masked (`3`), folder (`0`/`256`), paper (`1584`), filter (`4098`).

### 3. Raster Rendering (production quality)
- **Tile format**: Per 256Ã—256 tile = `256Â²` bytes alpha + `256Â²Ã—4` bytes BGRA, zlib-compressed.
- **Color types**: full color, grayscale, 1-bit monochrome.
- **Compositor**: Walks the layer hierarchy bottom-up, applies masks, clipping groups, blend modes, transforms, and paper color.

### 4. Blend Modes (fully mapped)
All 27+ CSP blend modes implemented with numpy vectorization: Normal, Multiply, Screen, Overlay, Soft/Hard/Vivid/Linear/Pin Light, Color Dodge/Burn, Add/Add Glow, Subtract, Difference, Exclusion, Lighten, Darken, Divide, Hard Mix, Darker/Lighter Color, HSV-based Hue/Saturation/Color/Brightness, and internal CSP `0x201`/`0x206` blend calls.

### 5. Adjustment Layers (first-pass done)
Level correction, tone curves (B-spline LUT with CSP's compact 16-bit point format), HSL adjustment, color balance (shadows/midtones/highlights with luminosity preservation), brightness/contrast, gradient maps.

## Active Area: Native Vector Rendering

The main focus is matching CSP's native vector stroke rendering for saved vector layers (`0x2081` stroked curves). This is far more complex than simple bitmap fallbacks â€” it involves:

### What's Already Pixel-Exact
| Guard Sample | Status |
|---|---|
| `Vector_Baseline` | Pixel-exact |
| `Vector_FlowPressure_50` | Pixel-exact |
| `Vector_Flow_50` | Pixel-exact |
| `Vector_OpacityRandom_50` | Pixel-exact |
| `Vector_OpacityPressure` | Pixel-exact |

### What's Still Not Exact
- **`Vector_SizePressure`**: The blocker. `max=226`, `mean=0.024409`, `visible=151` residual pixels. Residual is concentrated in tail dabs around segment ~26.

### Native Path Facts (from IDA analysis)
From the `AI_MEMORY.md`:
- Effector `0x31` drives size via `primary * (1.0 + (amount1 - 1.0) * secondary)` with `amount1=1.5`
- `sub_142CB45F0` composes lane outputs; pass-through graphs come via `PWBrushStyleManager`
- `sub_1422D8550` writes effective size to both plot radius and sampler feedback
- `AutoIntervalType=2`, `Hardness=1.0` â†’ interval scalar `0.08`
- Hard no-pattern span uses `sqrt(radiusÂ² - dyÂ²) - 0.4`
- `PWBrushDraw+264` is the no-pattern local draw scale (confirmed `1.0` for saved full-size)

### Key Rejected Hypotheses (preserved in memory)
- Don't change `amount1` from 1.5
- Don't shrink radius, hard-span margin, or interval values without native evidence
- Don't add duplicate-center or alpha-accumulation skip rules
- `PWBrushDraw+264` is NOT a radius scale â€” IDA maps the saved-route setter to `+272`
- Don't add hard spacing floors or reinterpret `StyleFlag&0x40`

### Fallback Rendering
For non-native vector content: comic frames, text balloons via approximate ellipse/rectangle rendering with tuning constants.

## Verification Workflow

`verify_one_clip.py` renders a `.clip` and compares against a ground-truth `.png` export:
```powershell
python verify_one_clip.py img/Vector_SizePressure.clip
```

The guard set for vector work is Baseline, Flow, FlowPressure, OpacityRandom, OpacityPressure, and SizePressure.
